Chapter 7: Software Engineering

Term

Definition
Software engineering Software engineering is the disciplined application of engineering principles to design, develop, and maintain software systems.
Integrated development environment (IDE) IDE's combine tools for developing software into a single, integrated package.


Association for Computing Machinery (ACM)
Purpose of ACM ACM's purpose is to advance computing as a science and profession, fostering collaboration and knowledge dissemination within the global computing community.
Benefits of ACM membership ACM membership provides access to top publications, conferences, networking, and career development resources for staying updated in the field of computing.
Student ACM membership ACM provides student memberships with access to resources like publications, conferences, and career development to support students in the field of computing.


Institute of Electrical and Electronics Engineers (IEEE)
Purpose of IEEE IEEE exists to advance technology for the benefit of humanity by fostering innovation and collaboration in electrical engineering and related fields.
Benefits of IEEE membership IEEE membership provides access to technical content, networking, career resources, and exclusive publications for professionals in electrical engineering and related fields.
Student IEEE membership IEEE offers student memberships, providing students access to resources and networking opportunities in electrical engineering and related fields.


Software Life Cycle: Development phase
Requirments analysis Analysis used to specify what services the proposed system wil provide, to identify any conditions on those services, and to define how the outside world will interact with the system.
Design Development involing the creation of a plan for the construction of the proposed system.
Implementation Development involing the actual writing of programs, creation of data files, and development of databases.
Testing Phase of development invilving the debugging of programs and confirming that the final software product was compatible with the software requirements row.


Modular implementation Modular implementation is a software design approach where a system is organized into independent and reusable modules, each encapsulating specific functionalities and interacting through well-defined interfaces.
Unified Modeling Language (UML) UML is a standardized visual language for modeling and documenting software systems using a set of symbols and notations.
Use case A use case in an interaction between the system and its users.
Use case diagram A diagram depicting the use cases within a software system.


Data relationships:
One-to-one When one entity communicates to another
One-to-many When one entity communicates to many others
Many-to-many When many entities communicates to many entities


Design pattern A predeveloped model for solving a recurring problem in software design.
Software Quality Assurance (SQA) A software quality standard, utilized to enforce quality control systems.


Software testing:
Glass box testing The software tester is aware of the interior structure of the software and uses this knowledge when designing the test.
Black box testing Tests that do not rely on knowledge of the software's interior composition.
Beta testing A preliminary version of software which is given to a segment of the intended audience with the goal of learning how the software performs in real-life situations before the final version of the product is solidified and released to the market.
Alpha testing Testing in a simmilar vein to beta testing performed at the developer's site.


Documentation:
User documentation Documentation used to explain te features of the software and describe how to use them.
System documentation Documentation used to describe the software's internal composition so that the software cand maintained later in its life cycle.
Technical documentation Documentation used to describe how a software system should be installed and serviced.


Intelectual property law
Software license The rights of the developwer formally expressed in legal terms. An agreement between the owner and user of the software product.
Software disclaimer Disclaimers present in a software license renouncing liablility from the producer.
Best defenses against software liablility claims (see last sentence in chapter) Apply sound software engineering principles during the software's development, to use a level of care compatible with the software's application, and to produce and maintain records that vailidate these endevors.
